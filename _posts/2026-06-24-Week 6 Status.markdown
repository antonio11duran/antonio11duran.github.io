This post is a part of a series of weekly status updates for an Android app project I am doing as a part of the Professional Development in Computer Science class at the University of Colorado Boulder.

## What did you do last week?

Last week I implemented the plugin secret gradle to hide my API key once I push my project to GitHub. The actual code was easy to implement, but I ran into issues when building and adding dependencies. Once I rebuilt the build, the correct BuildConfig file was generated to access the now secret API key.

## What do you plan to do this week?

I couldn't hit my goal of only displaying one review at a time, but now that I have the API key hidden, I can concentrate on just displaying the one review this week.

## Are there any impediments in your way?

The sheer amount of data in the first JSON kept causing my andriod emulator on my PC to crash the review app. I now have to use my phone to install the application, which is a bit more distracting, but it does run quicker than the emulator through Android Studio.

## Reflection on the process you used last week, how can you make the process work better?

I did a much better job this week with allocating large chunks of time to work on the project. I was able to dedicate 2-4 large stretches of time where I could work comfortably and be dedicated to my goals. Now that I have realized that my PC may not be powerful enough to host the emulator along with other applications, I can use an old Android device to host the reviews app for quicker/hands-on testing.
